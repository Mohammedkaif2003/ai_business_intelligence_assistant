 STEP 8: BUG FIX — MULTIINDEX CRASHES (TWO FILES)
================================================================================

ERROR:
   TypeError: Setting a MultiIndex dtype to anything other than object
   File 1: modules/insight_engine.py, line 36
   File 2: modules/executive_summary.py, line 36

ROOT CAUSE:
   When AI generates code like df.groupby(["Region", "Product"]).sum(),
   the result has a MultiIndex. Calling df.index.astype(str) on a
   MultiIndex throws a TypeError.

FIX (applied to BOTH files):
   1. Added at the top:
      if isinstance(df.index, pd.MultiIndex):
          df = df.reset_index()
      This flattens the MultiIndex into regular columns.

   2. Changed the fallback Entity assignment from:
      df["Entity"] = df.index.astype(str)
      to:
      df["Entity"] = [str(x) for x in df.index]
      This works with ANY index type.

 STEP 11: BUG FIX — AI GENERATING IMPORT STATEMENTS
================================================================================

ERROR:
   "Unsafe code detected: import"

ROOT CAUSE:
   Despite the AI prompt saying "Do NOT import libraries", the LLaMA model
   sometimes generated lines like "import pandas as pd" at the top of its
   code. The security scanner in code_executor.py correctly blocked this,
   but it prevented the actual analysis code from running.

FIX:
   Updated modules/ai_code_generator.py to STRIP import lines from the
   AI's response before passing to the executor:

   Before (simple filter):
   if not line.lower().startswith(("here", "sure", "this code")):
       code_lines.append(line)

   After (comprehensive filter):
   if stripped_line.startswith("import ") or stripped_line.startswith("from "):
       continue  # Skip import lines
   if stripped_line.lower().startswith(("here", "sure", "this code", "hope this")):
       continue  # Skip conversational text

   This way the security executor stays strict, but innocent imports from
   the AI are silently removed before they can trigger the block.

 STEP 12: BUG FIX — nlargest() MISSING ARGUMENT
================================================================================

ERROR:
   DataFrame.nlargest() missing 1 required positional argument: 'columns'

ROOT CAUSE:
   AI generated code like df.nlargest(3) on a DataFrame, but pandas
   requires: df.nlargest(3, columns='Revenue')

FIX:
   Added explicit rules to the AI prompt in ai_code_generator.py:
   - Rule 9: "If using nlargest() on a DataFrame, you MUST specify the
     'columns' argument"
   - Rule 10: "If the question asks for predictions, use np.polyfit
     since we have numpy imported as np"
